# Shipsticks_QA
The Playwright test automates the Step 1 happy path from the booking widget: it opens the ShipSticks staging homepage, verifies the origin/destination comboboxes and the Get started button are present, selects One way, fills the exact origin and destination addresses from the prompt, and clicks Get started using role‑based locators for stability. In the current staging environment, clicking Get started does not consistently navigate to a visible Step 1 screen or address summary, so I limited assertions to the homepage widget and left the stronger Step 1 checks (URL contains /step-1, Step 1 indicator, origin/destination, Ground, and ability to continue) described here for a stable environment.

## Playwright test

I implemented a Playwright test for the Step 1 booking happy path. It opens the ShipSticks staging homepage, verifies the booking widget (origin and destination comboboxes and the Get started button), selects "One way", fills the exact origin and destination addresses from the challenge, and clicks Get started using role-based locators. In the current staging environment, clicking Get started does not consistently navigate to a visible Step 1 screen or address summary, so I limited assertions to the booking widget and described the extra Step 1 checks (URL contains `/step-1`, Step 1 indicator, address summary, Ground selected, and Continue button) that I would add in a stable environment.
